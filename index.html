시연영상

세부기능 소개

귀찮게는 귀찮아서 만들었기 때문에, 간단하고 편리합니다.

    모아서 보기

  • 게시판 설정 탭에서 모아볼 게시판만 설정하면, 나만의 게시판 리스트가 생깁니다.

  • 더 이상 사이트를 옮겨다닐 필요없이, 두번의 동작으로 게시판들을 볼 수 있습니다.

  • 각 게시판 방에서는 키워드 알람과 북마크 설정이 가능합니다.

    나만의 게시판

  • 기본적으로 다양한 게시판을 준비할 예정입니다.

  • 만약 원하는 게시판이 없으면 게시판 이름과 주소를 입력해주세요.

  • 개발자가 게시판 작업을 마치면 알림을 보내드립니다. 한 유저만을 위한 게시판을 별도의 어플리케이션 업데이트 없이 추가시켜드립니다.

    북마크

  • 게시판 글을 보다가 맘에 드는 글, 다시 보고 싶은 글이 있으면 북마크 버튼만 꾹 눌러주세요.

  • 북마크 된 글은 북마크 탭에서 언제든지 확인 할 수 있습니다.

    키워드 알람

  • 알림을 받고 싶은 주제가 있으면 키워드를 등록해주세요.

  • 키워드가 포함된 글이 올라오면 바로 알람을 알려드립니다.

  • 게시판마다 다른 키워드를 입력해 놓을 수도 있습니다.

기술고민

개발하면서 했던 고민들

기능들을 구현하기 위해, 게시판 정보들을 어떻게 얻을 것인가?

- 외부DB에 정리되어있는 데이터를 가져다 쓰기보단 내 프로그램을 위한 데이터
- 서버에서 크롤러를 두고, 내 앱에 맞게 정보를 긁어서 정리(python, django, beautifulSoup)


어느 사이트에서나 적용될 만한 정형화된 틀을 갖춘 크롤러 만들기

- 1주일 간의 공부 후 실패, 개발자가 게시판에 맞춰 크롤러를 만들고, 정보 제공해주는 방식으로 선회


개개인 유저들의 게시판 요청을 어떻게 처리할 것인가?

- 공통의 리스트를 제공해주면, "나만의"라는 느낌이 살지 않는다.
- 유저의 요청에 따라 비활성화된 게시판을 생성하고, 개발자가 서버에 게시판 정보 리스트를 생성하면,
- 유저가 앱실행시에, retrofit을 통해 게시판정보를 받아와서 매칭 후 활성화 시키는 방식 적용
- 문제점 : 앱 삭제 이후에 다시 키면 개인은 리셋, 서버는 쓰레기정보 => 결국 차후에 회원기능을 추가하기로 함.
